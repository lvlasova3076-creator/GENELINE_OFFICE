# Протокол сохранения результатов сессии

## Правило — файл до коммита

Любой результат работы (аудит, отчёт, анализ, план) ОБЯЗАН быть создан как файл ПЕРЕД коммитом.
Давать результаты только в чате — запрещено.

## Структура сохранения

| Что создано | Куда сохранять |
|-------------|---------------|
| Аудит Instagram / контент | reports/content/ |
| Разведка рынка (scout_market) | reports/scout_market/ |
| Ежедневная сводка | reports/daily_brief/ |
| Исследование темы | reports/research/ |
| Проверка текста | reports/qa/ |
| Контент-план | reports/smm_calendar/ |

## Формат имени файла
`YYYY-MM-DD.md` или `YYYY-MM-DD_тема.md`

## Чеклист конца сессии

- [ ] Все результаты сохранены в файлы (не только в чат)
- [ ] HANDOFF.md обновлён
- [ ] git add конкретные файлы (не -A)
- [ ] git commit с описательным сообщением
- [ ] Лариса видит список что войдёт в коммит и подтверждает

## Почему это важно

Claude Code не хранит историю чата между сессиями.
Если результат не записан в файл → он исчезает при закрытии сессии навсегда.
Единственная страховка — файлы в git-репозитории.
