# /daily_brief — Утренняя сводка

## Назначение
Генерирует утреннюю сводку для начала рабочего дня. Показывает фокус, задачи, прогресс и предупреждает о рисках.

## Когда запускать
- В начале рабочего дня
- Клиент говорит: "сводка", "что сегодня", "план на день", "brief"

## Входные данные
| Файл | Зачем |
|------|-------|
| HANDOFF.md | Что было в прошлой сессии, следующий шаг |
| PROFILE.md | Приоритеты, стиль работы |
| COMPANY.md | Цели компании, текущие проекты |
| projects/ | Статус активных проектов (если есть) |

## Формат сводки

```
ФОКУС ДНЯ: [одна фраза — на чём сегодня]

ВЧЕРА: [что сделано в прошлой сессии, 2-3 строки из HANDOFF.md]

ТОП-3 ДЕЙСТВИЯ:
1. [конкретный шаг]
2. [конкретный шаг]
3. [конкретный шаг]

ПРОГРЕСС: [фаза проекта, % завершения, до дедлайна N дней — если есть]

⚠️ ПРЕДУПРЕЖДЕНИЯ: [дедлайны, отставание, потеря фокуса — если есть]
```

## Контекстные правила по дням
- **Понедельник** — добавить недельную сводку + приоритеты на неделю
- **Пятница** — итоги недели: что сделано / что нет / что перенести
- **Выходные** — укороченный формат (только фокус + следующий шаг)

## Правила
- Язык клиента (из PROFILE.md)
- Кратко (до 250 слов), без воды
- Без похвалы и мотивационных фраз
- Конкретные действия, не абстракции
- Проактивно предупреждать: "отстаёшь от плана" или "темп ок"
- Если клиент бросает задачу на полпути — напомнить: "сначала закончи, потом новое"

## Результат
Сохранить в `reports/daily_brief/YYYY-MM-DD.md`

## Цепочки
- После daily_brief клиент может запустить любой другой скилл по задачам дня
- scout_market → daily_brief (тренды попадают в сводку)
